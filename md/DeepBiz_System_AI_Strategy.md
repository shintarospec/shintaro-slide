#### Google Maps API詳細

| 項目 | 旧API（2024年まで） | 新API（2025年3月〜） |
|------|-------------------|---------------------|
| **使用API** | Text Search API | **Essentials API（推奨）** |
| **単価** | $0.032/検索 | **$0.005/検索** |
| **無料枠** | なし | **10,000件/月** |
| **コスト例（1,000件）** | $32 | **$0（無料枠内）** |
| **10,000件/月** | $320 | **$0（完全無料）** |
| **必要性** | ✅ 必須（店舗名→Place ID変換に不可欠） | ✅ 必須 |

**📌 重要**: 
- **2025年3月以降は月10,000件まで完全無料**
- 本資料の旧API料金（$0.032）は、**無料枠を考慮していない最大コスト**です
- 実運用では**Essentials APIの無料枠を活用**することで、**月10,000件までAPIコスト$0**

**コスト比較（月1,000件の場合）**:
```
旧API: 1,000件 × $0.032 = $32 (¥5,120)
新API: 1,000件 × $0.005 = $5 → でも無料枠内なので $0
```

#### なぜGoogle Maps APIが必要か？

```
【課題】
店舗名だけでは、Google Maps上の店舗を一意に特定できない
↓
【解決】
Google Maps Text Search APIで「Place ID」を取得
→ Place IDがあれば、その店舗のすべての情報にアクセス可能

例）
入力: "サロン名 東京 渋谷"
  ↓ Google Maps API
出力: Place ID "ChIJ5dj1n6yMGGARdYw5mEKBSqI"
  ↓
このPlace IDで、CID、Website、評価などすべて取得可能
```

## 🧠 企業分析フロー詳細（Phase 2）

### なぜPhase 2だけAIが必要なのか？

| 比較項目 | Phase 1（データ収集） | Phase 2（企業分析） |
|---------|---------------------|-------------------|
| **データ構造** | 構造化（JSON、HTML DOM） | 非構造化（自然言語テキスト） |
| **パターン** | 固定的（URL、メールアドレス） | 可変的（事業内容の表現方法） |
| **文脈理解** | 不要 | **必須**（「革新的」→「強み」） |
| **抽出対象** | 明示的（href属性、@マーク） | **暗黙的**（文章から推測） |
| **従来手法** | 正規表現・DOM解析で対応可能 | **ルールベースでは限界** |

## 📈 コスト最適化戦略

DeepBizでは**3つの最適化戦略**でコストを最小化：

### 1. Google Maps API無料枠の活用（Phase 1）

| 項目 | 効果 |
|------|------|
| **Essentials API使用** | 月10,000件まで完全無料（2025年3月〜） |
| **重複検索の排除** | Place IDをDB保存 → 2回目以降$0 |
| **バッチ処理** | Nearby Search APIで複数件を一括取得 |

### 2. AIキャッシュ戦略（Phase 2）

```
初回: AI解析 (0.023円) → DB保存（90日間）
2回目以降: DB取得（0円）← 完全無料
```

**効果**: 同じ企業の2回目以降のアクセスで**100%コスト削減**

### 3. Gemini 2.5 Flash-Lite選定理由（2026年版）

| AI モデル | 1社コスト | DeepBizとの差 |
|----------|----------|------------|
| GPT-4o | 3.04円 | **25倍高い** |
| Claude 3.5 Haiku | 1.06円 | 8.8倍高い |
| GPT-4o mini | 0.19円 | 1.5倍高い |
| **Gemini 2.5 Flash-Lite** | **0.12円** | ✅ **最安** |

**選定理由**: 最安コスト + 高速スループット + 2026年最新モデル

#### 推奨スペック：4G（石狩リージョン：¥3,227/月）

**選定理由**:
```
必要リソース:
├─ Selenium/ChromeDriver: 500MB〜1GB（並列実行時は×N）
├─ Flask + Gunicorn: 200MB〜400MB
├─ SQLite: 50MB〜100MB（データベース軽量）
└─ OS + バッファ: 1GB〜1.5GB

✅ 4G = 安定運用に最適
   - 同時2〜3タスクの並列スクレイピング可能
   - メモリ使用率60〜70%で余裕あり

△ 2G = ギリギリ動作
   - 単一タスクのみ
   - メモリ不足でクラッシュリスク

❌ 1G = 不安定
   - Selenium起動時にOOM（Out of Memory）エラー頻発
```
